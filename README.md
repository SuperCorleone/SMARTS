# SMARTS Replication Package

This repository contains the code, data, logs, and plotting scripts used to run the empirical evaluation for SMARTS, a stacking-based model-uncertainty handling approach for self-adaptive systems.

The package is organized as a practical replication workspace rather than a library release: the main entry points are experiment runners (`run_rq1.py`, `run_rq2.py`, `run_rq3.py`), plotting scripts (`plot_results.py`), and SLURM submission wrappers (`slurm_submit*.sh`).

## Repository structure

- `data/`
  - Raw experiment datasets.
  - `training_rescueRobot_450.csv` and `validation_rescueRobot_450.csv` are used by the drift and adaptation experiments.
  - `training_rescueRobot_25600_64.csv` is used by the RQ3 cost benchmark.
- `logs/`
  - JSON and TSV outputs generated by the experiment runners.
- `plots/`
  - Figures generated by `plot_results.py`.
- `paper_figures/`
  - Notebook-oriented helper code and a plotting notebook for paper-ready figures.
- `run_rq1.py`
  - RQ1: drift-only prequential prediction-quality experiment.
- `run_rq2.py`
  - RQ2: adaptation-decision quality experiment.
- `run_rq3.py`
  - RQ3: computational cost benchmark.
  - `--phase fit` runs one-shot fit cost.
  - `--phase online` runs online-update cost.
- `plot_results.py`
  - Generates publication-style plots from `logs/`.
- `slurm_submit.sh`
  - Top-level SLURM DAG for the main experiment suite.
- `slurm_submit_rq1.sh`, `slurm_submit_rq2.sh`, `slurm_submit_rq3.sh`
  - Focused SLURM entry points for individual research questions.

## Requirements

The codebase expects Python 3 and the following Python packages:

- `numpy`
- `pandas`
- `statsmodels`
- `scikit-learn`
- `mpmath`
- `geneticalgorithm`
- `matplotlib`

Install them with:

```bash
pip install numpy pandas statsmodels scikit-learn mpmath geneticalgorithm matplotlib
```

If you want to run the R-based cost baselines used in RQ3, you also need:

- `Rscript`
- R packages: `BAS`, `BMA`, `MASS`, `argparse`, `R.utils`, `ggplot2`

## Running the experiments

### RQ1: Drift-only prediction quality

RQ1 evaluates prequential precision, recall, F1, and Brier score under drift on the validation stream.

Example:

```bash
python run_rq1.py --drift-scenario 1x --drift-threshold 0.035
python run_rq1.py --drift-scenario 6x --drift-threshold 0.035
```

Outputs:

- `logs/rq1_A_th{TH}_drift{SC}.json`
- `logs/rq1_A_th{TH}_drift{SC}_summary.tsv`

### RQ2: Adaptation decision quality

RQ2 compares the quality of GA-guided adaptation decisions under model uncertainty.

Single-node example:

```bash
python run_rq2.py --drift-scenario 6x --oracle logit --oracle-label-source match-drift
```

Distributed execution is phase-based:

```bash
python run_rq2.py --phase setup --drift-scenario 6x
python run_rq2.py --phase online --drift-scenario 6x --drift-threshold 0.035
python run_rq2.py --phase online_ga --drift-scenario 6x --drift-threshold 0.035 --oracle logit --chunk 0 --n-chunks 10
python run_rq2.py --phase bma_online --drift-scenario 6x --retrain-every 100
python run_rq2.py --phase bma_online_ga --drift-scenario 6x --drift-threshold 0.035 --oracle logit --chunk 0 --n-chunks 10
python run_rq2.py --phase best_logit --drift-scenario 6x --oracle logit --chunk 0 --n-chunks 10
python run_rq2.py --phase merge --drift-scenario 6x --drift-threshold 0.035 --oracle logit
```

Outputs:

- `logs/rq2_A_drift{DR}_th{TH}_oracle{O}_oraclelabels{SRC}.json`
- `logs/rq2_A_drift{DR}_th{TH}_oracle{O}_oraclelabels{SRC}_summary.tsv`

### RQ3: Computational cost

RQ3 has two phases.

#### RQ3-A: one-shot fit cost

```bash
python run_rq3.py --phase fit --method best_logit --n-runs 1
```

Outputs:

- `logs/rq3_best_logit.tsv`

#### RQ3-B: online-update cost

Grid mode measures amortized online-update latency on the synthetic `(samples, vars)` grid.

```bash
python run_rq3.py --phase online --online-mode grid --drift-scenario 6x --retrain-every 100
```

Chunked execution:

```bash
python run_rq3.py --phase online --online-mode grid --drift-scenario 6x --chunk 0 --n-chunks 16 --retrain-every 100
python run_rq3.py --phase online --online-mode grid --drift-scenario 6x --merge --retrain-every 100
```

Legacy mode keeps the old 450-row drift stream benchmark:

```bash
python run_rq3.py --phase online --online-mode legacy --drift-scenario 6x --retrain-every 1
```

Outputs:

- grid: `logs/rq3_online_grid_drift{DR}_re{K}.tsv`
- legacy: `logs/rq3_online_drift{DR}.tsv`

## Generating plots

Generate all plots:

```bash
python plot_results.py
```

Or per research question:

```bash
python plot_results.py --rq1-only
python plot_results.py --rq2-only
python plot_results.py --rq3-only
```

The main plot outputs are written to `plots/`.

## Running with SLURM

Run the full suite:

```bash
bash slurm_submit.sh
```

Run only one research question:

```bash
bash slurm_submit_rq1.sh
bash slurm_submit_rq2.sh
bash slurm_submit_rq3.sh
```

Examples:

```bash
bash slurm_submit_rq1.sh --thresholds "0.035" --drift-scenarios "1x 3x 6x"
bash slurm_submit_rq2.sh --rq2-drifts "1x 3x 6x" --oracles "logit"
bash slurm_submit_rq3.sh --rq3-chunks 16 --rq3-online-chunks 16 --time-rq3a 4:00:00 --time-rq3b 4:00:00
```

## Notes

- RQ2 and RQ3 can take a long time to run, especially on larger grids.
- RQ3 includes timeout guards per cell, but wall-clock time still depends heavily on chunking and scheduler parallelism.
- `plot_results.py` can read both current outputs and some legacy file names to ease migration.
